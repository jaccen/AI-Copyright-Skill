# AI项目软件著作权登记特殊指南

## 1. AI项目类型识别

### 1.1 类型判定决策树

```
项目入口文件是什么？
├── app.py / main.py / serve.py → AI服务类
│   ├── 含 FastAPI/Flask/Gradio → Web API服务
│   ├── 含 Streamlit/Dash → 交互式Demo
│   └── 含 gRPC/Thrift → 微服务
├── train.py / trainer.py → AI训练类
│   ├── 含 torch.nn / tf.keras → 模型训练
│   ├── 含 accelerate/deepspeed → 分布式训练
│   └── 含 transformers/peft → 预训练/微调
├── inference.py / predict.py → AI推理类
│   ├── 含 ONNX/TensorRT → 模型优化部署
│   ├── 含 triton/venv → 推理服务
│   └── 含 openvino/coreml → 边缘部署
├── pipeline.py / workflow.py → AI流水线类
│   ├── 含 airflow/kubeflow → 编排调度
│   ├── 含 langchain/crewai → Agent应用
│   └── 含 mmdet/ultralytics → 检测流水线
└── package.json → 前端/全栈类
    ├── 含 three.js/opencv.js → 可视化/渲染
    ├──含 electron/tauri → 桌面应用
    └── 含 next.js/nuxt.js → Web应用
```

### 1.2 各类型软著说明书侧重点

| 项目类型 | 功能说明重点 | 截图建议 |
|----------|-------------|----------|
| Web API服务 | 接口列表、请求/响应格式、鉴权方式、限流策略 | Swagger文档页、Postman测试、监控面板 |
| 交互式Demo | 交互方式、输入/输出说明、参数调节、结果展示 | 主界面、参数调节面板、结果对比页 |
| 模型训练框架 | 训练配置、超参数管理、实验跟踪、模型版本 | 训练仪表盘、实验对比页、配置界面 |
| 推理服务 | 模型加载、批处理策略、性能指标、健康检查 | 性能监控、模型管理界面 |
| Agent应用 | 工具调用链、记忆管理、多轮对话、知识检索 | 对话界面、工具调用日志、知识库管理 |
| 检测流水线 | 数据输入、预处理、检测/识别、结果输出 | 检测结果可视化、Pipeline配置页 |

## 2. AI源代码文档特殊处理

### 2.1 核心源文件优先级

| 优先级 | 文件类型 | 推荐扫描模式 |
|--------|----------|--------------|
| 1 | 模型定义(model.py/network.py/arch.py) | 必选 |
| 2 | 训练入口(train.py/trainer.py) | 必选 |
| 3 | 推理入口(inference.py/predict.py) | 必选 |
| 4 | 数据处理(dataset.py/dataloader.py) | 推荐 |
| 5 | 损失函数(loss.py/criterion.py) | 推荐 |
| 6 | 配置文件(config.yaml/hparams.py) | 可选 |
| 7 | 工具函数(utils.py/helpers.py) | 可选 |
| 8 | 测试文件(test_*.py) | 不选 |

### 2.2 源代码脱敏清理清单

- [ ] 删除所有API Key、Token、Secret（含环境变量引用如 `os.environ["OPENAI_API_KEY"]`）
- [ ] 删除数据集路径中的绝对路径（替换为 `./data/`）
- [ ] 删除内网地址和域名
- [ ] 删除个人邮箱、姓名
- [ ] 保留注释中关键算法说明（审查员可能参考）

### 2.3 代码行数不足60页（3000行）的处理策略

1. **正常情况**：不足60页的全部提交
2. **代码稀缺**（<30页）：
   - 优先补充工具脚本、配置文件
   - 包含README中的系统架构说明（作为补充文档）
   - 含前端代码（HTML/CSS/JS）一并纳入
3. **代码超量**（>3000行）：
   - 按优先级表选前1500行 + 后1500行
   - 确保「前30页」包含模型定义和核心算法
   - 确保「后30页」包含推理入口和配置

## 3. AI软著说明书模板

### 3.1 推荐章节结构

```
1. 引言
   1.1 编写目的
   1.2 软件概述
       - 软件名称及版本
       - 功能定位（说明是AI推理服务/训练平台/Agent应用等）
       - 核心AI能力描述（面向审查员，避免专业术语）
   1.3 运行环境
       - 硬件要求（含GPU型号及显存要求）
       - 软件要求（Python版本、深度学习框架版本）
       - 网络环境

2. 软件安装与初始化
   2.1 环境配置（含CUDA/驱动版本）
   2.2 依赖安装
   2.3 模型权重获取
   2.4 配置文件说明

3. 功能说明
   3.1 功能模块总览
   3.2 AI核心引擎模块
       - 模型加载与初始化
       - 推理/预测流程
       - 输出后处理
   3.3 数据处理模块
       - 输入数据预处理
       - 数据增强与验证
   3.4 服务接口模块
       - API接口列表
       - 请求/响应格式
   3.5 管理与监控模块（如有）
       - 性能监控
       - 模型版本管理

4. 非功能需求
   4.1 性能指标（推理延迟/吞吐量/精度）
   4.2 安全性
   4.3 可扩展性

5. 常见问题
```

## 4. AI软著常见问题与避坑

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| "纯算法不可软著"误区 | 软著保护的是代码表达形式而非思想 | 强调"软件"而非"算法"，描述代码实现 |
| 开源代码混用 | 项目使用了第三方开源库 | 源代码文档中明确标注自研代码范围 |
| 预训练模型引用 | 使用了开源预训练权重 | 软著声明保护范围不包含预训练权重本身 |
| 文档过于技术化 | 审查员可能不熟悉AI术语 | 功能描述面向非技术人员，避免数学公式 |
| 缺少人工操作步骤 | 软著要求"人机交互过程" | 即使是服务端程序也需描述部署/配置/监控过程 |
