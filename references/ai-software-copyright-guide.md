# AI项目软件著作权登记特殊指南

## 1. AI项目类型识别

### 1.1 类型判定决策树

```
项目入口文件是什么？
├── app.py / main.py / serve.py → AI服务类
│   ├── 含 FastAPI/Flask/Gradio → Web API服务
│   ├── 含 Streamlit/Dash → 交互式Demo
│   ├── 含 gRPC/Thrift → 微服务
│   └── 含 docker/Dockerfile → 容器化部署
├── train.py / trainer.py → AI训练类
│   ├── 含 torch.nn / tf.keras → 模型训练
│   ├── 含 accelerate/deepspeed → 分布式训练
│   ├── 含 transformers/peft → 预训练/微调
│   └── 含 trl/rlhf → 对齐训练
├── inference.py / predict.py → AI推理类
│   ├── 含 ONNX/TensorRT → 模型优化部署
│   ├── 含 triton/venv → 推理服务
│   ├── 含 openvino/coreml → 边缘部署
│   └── 含 tflite/ncnn/ONNX → 端侧部署
├── render.py / splat.py / gaussian.py / nerf.py → 3D视觉类
│   ├── 含 gsplat/nerfstudio → 三维重建/渲染
│   ├── 含 open3d/trimesh → 点云/网格处理
│   ├── 含 colmap/theia → SFM/SLAM
│   └── 含 pytorch3d/nvdiffrast → 可微分渲染
├── pipeline.py / workflow.py → AI流水线类
│   ├── 含 airflow/kubeflow → 编排调度
│   ├── 含 langchain/crewai → Agent应用
│   ├── 含 mmdet/ultralytics → 检测流水线
│   └── 含 rag/chroma/faiss → RAG/检索增强
├── generate.py / sample.py / diffusion.py → 生成式AI类
│   ├── 含 diffusers/stable-diffusion → 文生图/视频
│   ├── 含 xformers/flash-attn → 高效注意力
│   └──含 controlnet/ip-adapter → 可控生成
├── robot.py / control.py / vla.py → 具身智能类
│   ├── 含 mujoco/isaac-sim → 仿真环境
│   ├── 含 ros2/robotnik → 机器人操作系统
│   └── 含 pybullet/drake → 物理仿真
└── package.json → 前端/全栈类
    ├── 含 three.js/opencv.js → 可视化/渲染
    ├── 含 electron/tauri → 桌面应用
    └── 含 next.js/nuxt.js → Web应用
```

## 2. AI源代码文档特殊处理

### 2.1 核心源文件优先级

| 优先级 | 文件类型 | 推荐扫描模式 |
|--------|----------|--------------|
| 1 | 模型定义(model.py/network.py/arch.py) | 必选 |
| 2 | 训练入口(train.py/trainer.py) | 必选 |
| 3 | 推理入口(inference.py/predict.py) | 必选 |
| 4 | 渲染/重建(render.py/splat.py/gaussian.py) | 3D视觉类必选 |
| 5 | 数据处理(dataset.py/dataloader.py) | 推荐 |
| 6 | 损失函数(loss.py/criterion.py) | 推荐 |
| 7 | 生成/采样(generate.py/sample.py/diffusion.py) | 生成式AI类必选 |
| 8 | 控制/策略(control.py/policy.py/vla.py) | 具身智能类必选 |
| 9 | 检索/索引(retriever.py/indexer.py/rag.py) | RAG类必选 |
| 10 | 配置文件(config.yaml/hparams.py) | 可选 |
| 11 | 工具函数(utils.py/helpers.py) | 可选 |
| 12 | 测试文件(test_*.py) | 不选 |

### 2.2 源代码脱敏清理清单

- [ ] 删除所有API Key、Token、Secret（含环境变量引用如 `os.environ["OPENAI_API_KEY"]`）
- [ ] 删除数据集路径中的绝对路径（替换为 `./data/`）
- [ ] 删除内网地址和域名
- [ ] 删除个人邮箱、姓名
- [ ] 删除具体硬件型号（A100/H100→GPU，Jetson→边缘计算设备）
- [ ] 删除云服务端点URL（替换为 `https://api.example.com/`）
- [ ] 删除数据库连接串中的密码
- [ ] 保留注释中关键算法说明（审查员可能参考）

### 2.3 代码行数不足60页（3000行）的处理策略

1. **正常情况**：不足60页的全部提交
2. **代码稀缺**（<30页）：
   - 优先补充工具脚本、配置文件
   - 包含README中的系统架构说明（作为补充文档）
   - 含前端代码（HTML/CSS/JS）一并纳入
3. **代码超量**（>3000行）：
   - 按优先级表选前1500行 + 后1500行
   - 确保「前30页」包含模型定义和核心算法
   - 确保「后30页」包含推理入口和配置
   - 3D视觉类确保「前30页」包含模型定义和渲染代码
   - 生成式AI类确保「前30页」包含模型定义和采样/生成代码

## 3. AI软著说明书模板

### 3.1 推荐章节结构（通用型）

```
1. 引言
   1.1 编写目的
   1.2 软件概述
       - 软件名称及版本
       - 功能定位（说明是AI推理服务/训练平台/Agent应用等）
       - 核心AI能力描述（面向审查员，避免专业术语）
   1.3 运行环境
       - 硬件要求（含GPU型号及显存要求）
       - 软件要求（Python版本、深度学习框架版本）
       - 网络环境

2. 软件安装与初始化
   2.1 环境配置（含CUDA/驱动版本）
   2.2 依赖安装
   2.3 模型权重获取
   2.4 配置文件说明

3. 功能说明
   3.1 功能模块总览
   3.2 AI核心引擎模块
       - 模型加载与初始化
       - 推理/预测流程
       - 输出后处理
   3.3 数据处理模块
       - 输入数据预处理
       - 数据增强与验证
   3.4 服务接口模块
       - API接口列表
       - 请求/响应格式
   3.5 管理与监控模块（如有）
       - 性能监控
       - 模型版本管理

4. 非功能需求
   4.1 性能指标（推理延迟/吞吐量/精度）
   4.2 安全性
   4.3 可扩展性

5. 常见问题
```

## 4. AI软著常见问题与避坑

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| "纯算法不可软著"误区 | 软著保护的是代码表达形式而非思想 | 强调"软件"而非"算法"，描述代码实现 |
| 开源代码混用 | 项目使用了第三方开源库 | 源代码文档中明确标注自研代码范围 |
| 预训练模型引用 | 使用了开源预训练权重 | 软著声明保护范围不包含预训练权重本身 |
| 文档过于技术化 | 审查员可能不熟悉AI术语 | 功能描述面向非技术人员，避免数学公式 |
| 缺少人工操作步骤 | 软著要求"人机交互过程" | 即使是服务端程序也需描述部署/配置/监控过程 |
| 3D视觉软件缺少交互描述 | 3D重建渲染后若无人工操作描述，审查员可能认定缺少人机交互 | 补充视角选择、参数调节、模型编辑、导出等用户操作步骤 |
| 生成式AI缺少安全声明 | 审查员可能关注内容安全合规 | 增加"安全过滤"功能章节，描述输入/输出审核机制 |
| 具身智能缺少硬件描述 | 软著要求运行环境明确 | 详细描述传感器/执行器型号范围、通信协议、安全参数 |
| RAG系统知识库版权 | 知识库含第三方内容 | 声明知识库内容不在保护范围，仅保护检索增强生成软件 |
| 边缘部署交叉编译 | 源代码含宿主机和目标设备两套环境 | 源代码文档优先提交模型推理和端侧优化代码 |
