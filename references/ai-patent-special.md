# AI领域专利撰写特殊指南

## 1. 算法可专利性判定框架

### 1.1 中国国知局审查标准（2024修订指南第二部分第九章）

| 判定层级 | 通过条件 | 常见驳回理由 |
|----------|----------|--------------|
| 第一层：不属于智力活动的规则和方法 | 算法与技术手段结合解决技术问题 | 纯数学优化、业务流程自动化 |
| 第二层：具备技术特征 | 产生技术效果（速度提升、精度提高、资源节约） | 仅提高计算效率但无技术场景 |
| 第三层：充分的说明书公开 | 公开训练数据构成、模型结构、推理流程至可实施 | "黑箱"型公开不充分 |

### 1.2 AI专利撰写三要素

1. **技术问题**：必须锚定具体技术场景（如"三维场景重建中的实时渲染瓶颈"），不可是"效率低""精度差"等抽象表述
2. **技术手段**：将算法步骤与硬件/软件架构绑定（如"GPU并行计算单元执行XX算子"），每一算法步骤需关联系统组件
3. **技术效果**：量化技术效果（如"推理延迟从120ms降至35ms"），优于"效果显著提升"

### 1.3 独立权利要求AI模板

```
一种基于[技术场景]的[方法名称]方法，其特征在于，包括：

S1. [数据获取/预处理步骤]，通过[具体技术手段]获取[输入数据描述]；
S2. [核心算法步骤1]，利用[模型/网络名称]对所述[输入数据]进行[处理动作]，
    所述[模型/网络]包括[结构特征]，以[技术目的]；
S3. [核心算法步骤2]，基于S2的输出，通过[具体计算方式]确定[中间结果]；
S4. [输出/应用步骤]，根据所述[中间结果]生成[最终输出]，用于[技术应用场景]。

其中，所述[模型/网络]的训练过程包括：
T1. [训练数据构建步骤]；
T2. [损失函数与优化步骤]；
T3. [训练收敛条件]。
```

## 2. AI技术方案拆分布局策略

### 2.1 单发明vs分案申请判断

| 判断维度 | 单发明（一件申请） | 分案申请（多件） |
|----------|---------------------|-------------------|
| 模型架构+训练方法 | 核心创新在架构，训练方法是配套 | 架构和训练各有独立创新点 |
| 推理方法+部署方案 | 推理优化是架构的自然延伸 | 部署方案引入新的硬件/编排创新 |
| 数据处理+下游应用 | 应用是数据处理流程的终端 | 应用场景差异大（如医疗vs工业） |

### 2.2 推荐布局：3件系列申请

1. **核心架构/算法件**：保护模型结构、核心算法流程、损失函数设计
2. **训练方法件**：保护数据构建、采样策略、训练技巧、超参数搜索
3. **推理部署件**：保护模型压缩、量化策略、推理优化、服务架构

## 3. 六类论文到专利映射表

| 论文要素 | 专利映射 | 转换要点 |
|----------|----------|----------|
| Research Problem | 技术问题 | 从学术语言转为工程痛点描述 |
| Contribution | 技术方案 | 将"we propose"转为"包括...步骤" |
| Method Modules | 装置单元 | 每个module对应一个装置单元+方法步骤 |
| Algorithm Flow | 实施流程 | 伪代码→步骤化描述，附流程图 |
| Experimental Outcome | 技术效果 | 定量结果→"进一步地，本发明相比XX方法提升YY%" |
| Ablation Study | 从属权利要求 | 每个ablation变体→一条从属权利要求 |

## 5. AI领域脱敏特殊规则

| 原始内容 | 脱敏替换 | 说明 |
|----------|----------|------|
| 具体数据集名(Cityscapes/KITTI) | "预设道路场景数据集" | 保留数据集属性描述 |
| 模型具体参数量(7B/13B/70B) | "预设规模的预训练模型" | 用范围替代("千万至千亿级参数") |
| 硬件型号(A100/H100) | "图形处理器" | 保留计算能力描述 |
| 训练时长/成本 | "预定训练周期" | 用相对描述替代绝对值 |
| 开源框架名(PyTorch/TF) | "深度学习框架" | 保持实现无关性 |
| 具体API/云服务名 | "远程服务接口" | 保留接口功能描述 |
